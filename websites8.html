Appealing website

MDN Web Docs

a. What is the URL of the website?
https://developer.mozilla.org/en-US/
b. Does this page use tables? If so, for what purpose?
The main page layout does not rely on HTML tables for design. Content is structured with semantic elements (header, nav, main, sections, articles). If tables appear elsewhere on MDN, they are used to present structured data (for example comparison charts or property lists in documentation), not to lay out the whole page.
c. Does this page use CSS? If so, for what purpose?
Yes, extensively. CSS is used for:
  • Page layout — responsive grid/flex sections, spacing, and alignment
  • Text and color — typography, headings, links, and a consistent dark theme
  • Another reason — accessibility (skip links, focus states) and visual hierarchy so featured articles stand out
d. Is this page appealing or unappealing? List three reasons.
Appealing.
  • Clear navigation and readable typography make it easy to find documentation.
  • Consistent colors and spacing create a professional, calm look.
  • Content is organized into logical sections (featured articles, news, contributions) instead of feeling cluttered.
e. If unappealing, what would you do to improve it?
Not applicable — this site is appealing. No major redesign is needed for this example.
Unappealing website

Arngren — Hobby Electronics

a. What is the URL of the website?
http://www.arngren.net/
b. Does this page use tables? If so, for what purpose?
Yes. The homepage uses HTML tables heavily for page layout. Product images and text are placed inside table cells in a dense grid, which was a common old technique before modern CSS layout (flexbox/grid).
c. Does this page use CSS? If so, for what purpose?
Very little compared to modern sites. Most layout comes from tables and inline/old HTML styling. There is minimal use of CSS for text and color configuration (basic fonts and link colors), but not for a flexible, responsive page layout.
d. Is this page appealing or unappealing? List three reasons.
Unappealing.
  • The page feels crowded: many small images and links compete for attention with no clear focal point.
  • Visual hierarchy is weak — it is hard to tell what is most important on the page.
  • The design looks outdated (table-based layout, inconsistent image sizes), which hurts usability on today’s screens.
e. If unappealing, what would you do to improve it?
  • Replace table-based layout with CSS Grid or Flexbox and a simple header + product grid.
  • Use larger, consistent product thumbnails with clear titles, prices, and one primary “Shop” or category menu.
  • Add whitespace, a limited color palette, and mobile-friendly responsive rules so the site is readable on phones and tablets.
